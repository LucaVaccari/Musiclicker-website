Big numbers in the game

As seen in the design part, the number of notes grows exponentially. A simple computer with at most 64 bit can't handle a number like 10^50. To fix this problem I created a custom class that stores the base number and the exponent of the power with base 10. That is number = baseNum * 10 ^ exponent.
The base number can only reach 999 and then it gets reduced and the exponent increased. I then created an array of suffixes (like "K", "M", "G", "T", ecc) to put at the end of the number to create a human readable string.
